The projects in this portfolio are my greatest accomplishments from my masters' studies. Unlike my published research, which developed within an established 
advising relationship and with relevant coursework as a foundation, this work grew out of entirely self-directed study. I acquired the necessary background, 
conducted literature reviews, then defined and pursued research questions independently. These three projects are the culmination of that process. Each 
addresses a distinct problem in performance engineering of hash-based data structures and hash functions. \\ 

\noindent
The first generalizes binary fuse filters, a recent approximate set membership structure, to support non-integer arity, the number of hash functions per query, 
natively. Binary fuse filters offer only a discrete tradeoff: increasing arity reduces memory overhead but increases query time. The standard approach to 
achieve a continuous tradeoff is to partition data between disjoint filters of different arity. Native non-integer arity make this tradeoff principled within a 
single construction and modestly reduces memory overhead over the partitioning approach. \\

\noindent
The second introduces the first optimal parameter tuning algorithm for secure learned Bloom filters, a recent approximate set membership structure that combines
a learned model with two secure filters. Since standard tuning algorithms do not apply, the recommended approach is simply to allocate memory evenly between the
two filters. This is a poor default: it cannot target an expected false positive rate, nor bound the worst-case rate, and can waste memory when either 
constraint is defined. I give the first exact tuning algorithm to minimize memory footprint subject to expected and maximum false positive rate constraints,
while maintaining security guarantees. \\

\noindent
The third extends entropy-aware hash function specialization, a technique that learns which bytes of a key carry the most information and hashes only those. 
Existing work assumes uniform key frequency and a generic hash table model, and does not generate efficient x64 implementations. I generalize the approach with 
new cost models that account for non-uniform key frequency and the specific structure of the target data structure, and add code generation targeting x64. This 
reduces both hash latency and hash table query latency. \\

\noindent
Together these projects move between theory and efficient implementation, driven by an experimental approach: data suggested new theory, and that theory 
motivated new experiments. Each required building fluency in an unfamiliar area, and each left open problems I continue to pursue. They are the clearest 
evidence that I have become a computer scientist.
